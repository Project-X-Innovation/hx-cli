## Description
How often do we make use of the research/report feature to plan a future build or fix? It would be nice to be able to just reference that report when creating a new ticket. I'd like to have a /implement command that lets you choose a previously completed research/report item and use the report of this previously completed report as the input for the new ticket. 

I think it's very clever. Probably the ticket will need some rewording. It'll need a prefix of "go ahead and implement the feature in this report." I'll let you brainstorm exactly how you want it to work out. Think about it a little. Think about all the things that can go wrong, all the mismatches that can happen. Think about how you would want to round this out, harden it, and make it really slick and user friendly and just work beautifully. 

But the idea is to very easily use an existing report which researched the feature as the content for a new ticket. I made a report to research it; now I want it implemented. My idea was just to make / implement reference to that report ticket and go for it. You can add some more text that also gets incorporated, so the prompt can be something like "Implement the plan below," and then followed by the rest of the text in the box, and then followed by the report. That's just one idea; you can brainstorm some other good options. Make it really nice and fun to use.
